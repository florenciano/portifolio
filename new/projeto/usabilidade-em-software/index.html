Usabilidade em software

Design de Interação | Pesquisa de usuário | Arquitetura de Informação | Prototipagem | Design de Interface

Este projeto foi realizado para a empresa de desenvolvimento de softwares desktops Acronus Software sediada em Campinas-SP, e esta oportunidade contribuiu para responder como que a ciência da UX poderia auxiliar na melhoria da usabilidade de seus produtos, uma vez que havia inúmeras insatisfações de seus usuários. O escopo deste projeto foi para o seu produto carro-chefe (até então) : Acronus System que é um sistema de gerenciamento e controle de acessos para rede de computadores.


Nov 2011 - Fev 2012

Análise heurística

O software apresenta apenas uma interface de usuário (imagem a seguir). As interações ocorrem dentro das limitações de cada painel agrupado na tela, sendo que o painel central (maior) é uma listagem dos computadores onde os usuários visualizam os status das operações realizadas.

 

Resumindo os problemas encontrados:

  • Barra de ferramentas: Set de iconografia com muitos detalhes e com qualidade duvidosa prejudicando a memorização do usuário e inviabilizando na utilização em tamanhos reduzidos. Ícones repetidos para funções distintas.
  • Arquitetura de informação confusa: As funcionalidades do software não apresenta um agrupamento organizado. Estas funções são distribuídas, à medida que são desenvolvidas, sem estratégia alguma de alocação.
  • Elementos de interface usados de forma errada: uso inadequado de componentes de tela para à função desempenhada.

Avaliando com usuários

Para descobrir quais seriam as melhores decisões a serem tomadas foi realizado o Teste de usabilidade com os principais usuários do software. Foi planejado a execução de tarefas corriqueiras a um grupo de 10 usuários. Para medir e compreender os feedbacks foi realizado um questionário de satisfação. Também houve a preocupação de avaliar a relevância do software e quais as reais necessidades, por isso, foi anexado aos usuários-testes uma pesquisa qualitativa. A seguir um resumo de alguns dos principais pontos de destaques.

Cardsorting

A partir das análises dos dados dos usuários o passo seguinte realizado foi o cardsorting para descobrir como os usuários organizam um conjunto de informações. Também serviu para definir as nomenclaturas e rótulos das funcionalidades do software. O resultado final é um protótipo de uma arquitetura da informação de um sistema. Para a realização desta etapa usou-se a ferramenta online WebSort (na época do teste o funcionamento da ferramenta limitava-se a 10 usuários/trial).

A seguir o resultado do Cardsorting representado em dendograma.

Auxílio na interpretação do dendograma: A identificação das relações entre dois itens no diagrama é visualizada através das separações das cores (branco e verde). Uma barra vertical une grupos que representam relações entre si. A leitura fica facilitada se analisada da direita para esquerda. À medida que visão ficar mais à direita teremos uma redução na similaridade entre os grupos, e quanto mais à esquerda a visão se concentrar, maior a similaridade, sendo que cada grupo deverá ter pelo menos dois itens agrupados para gerar relações.

Os nomes listados acima são rótulos dos comandos do software indicados pelos users-testes, e que portanto, representam com mais exatidão o modelo mental dos usuários.

Avaliando alternativas

A partir deste ponto começou a pensar e avaliar caminhos criativos. Para considerar as representações de design de interações iniciou-se a realização de prototipagem em papel. Dei prioridade a construção de uma sequência lógica coerente onde os usuários pudessem realizar suas tarefas com maior fluidez.

A definição da taxonomia

Uma vez testada e validada o fluxo de navegação e as interações obtidos na etapa da prototipagem definiu-se a nova arquitetura de informação que proporcionaria:

  • Ajustar a nomenclatura dos comandos para criar padronizações;
  • Proporcionar aos usuários affordances por meio das metáforas visuais;
  • Criar sistemas de notificacões de status das operações realizadas.

Criando a interface gráfica do usuário (GUI)

A prototipação continou na plataforma digital. O desafio enfrentado neste estágio foi aproximar a interface do software com o universo visual da marca e a preocupação em compor um layout que aprimorasse a ergonomia. Veja a seguir o resultado final proposto para a interface do software Acronus System.

Para auxiliar a empresa no seu desenvolvimento de software e para nortear se as adaptações das interfaces seguem o padrão de design desenvolvido foi fornecido um guia de interface de software uma espécie de guia de boas práticas. É uma garantia que a empresa estará sempre pensando na boa utilização para os usuários.
