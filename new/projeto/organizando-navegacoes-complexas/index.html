Organizando navegações complexas

Design de Interação | Arquitetura de informação | Design de Interface | Teste de usabilidade | Prototipagem

Nesta página você verá dois projetos que tratam de como uma organização na navegação é fundamental para que os usuários estejam aptos a realizar suas tarefas com maior facilidade. O primeiro é um projeto conceito, que surgiu depois de experimentar na própria pele como é difícil encontrar as informações desejadas num sistema desorganizado e com uma interface confusa, decidi que seria uma boa ideia propor alternativas de melhorias; o segundo também aborda o mesmo tema, só que dentro de uma arquitetura mais enxuta.

 

Por onde eu começo?

Se você, assim como eu já teve que utilizar os serviços do Detran.SP sabe que não é fácil saber por onde navegar com tantas opções na tela, né? A imagem a seguir mostra a interface do sistema na época do acesso.


Set 2017

Enumerei em 5 itens o complexo sistema de navegação junto com o mecanismo de busca para poder verificar como esta a disposição e organização das informações. 1) Uma simples pesquisa com o termo "multa" surgiram mais de 10 páginas de resultados com 20 respostas por página. Tanta informação que fiquei na dúvida de qual seria o contexto da minha necessidade; 2) Todas as informações contidas no site se resume as estas 4 opções. Um pouco vago, mas talvez seja a única coisa mais suscinta até aqui; 3) são serviços online que o sistema oferece, não fica claro para qual categoria se encaixa. São muitos os serviços e, consequentemente, recebe mais atenção do que qualquer outro elemento da interface; 4) Podemos deduzir que há serviços mais utilizados (ou mais relevantes) e que por isso ganharam maior destaque visual; 5) São mais opções de navegação da categoria. Não há consistência visual entre eles e você fica confuso se são partes de um todo ou se são coisas distintas entre si.

Sabemos que o problema da navegação estar complexa é porque são inseridas novas funções dentro de uma arquitetura de informação previamente não projetada para isto. A inconsistência visual é resultado de não seguir um guideline de desenvolvimento de interface. O Detran.SP preocupou-se em disponibilizar TODAS as informações ao público de uma única vez e o usuário teve que pagar o preço: não se acha nenhuma informação com facilidade!

Colocando ordem na casa

A ideia apresentada a partir daqui é uma tentativa de arrumar a bagunça, propor algo mais claro ao usuário. As ideias que compartilho aqui são uma resposta para a simples pergunta: O que você faria para melhorar este cenário?

Antes de sair abraçando o mundo vamos definir o escopo:

  • Partindo do pressuposto que as informações em maior destaque são os serviços mais procurados pelo público, então usaremos esta hipótese como parâmetro para projetar uma navegação funcional;
  • Eliminação de redundâncias! Serviços semelhantes ficarão agrupados e manterão um padrão de nomenclaturas;
  • Reduzir a carga de informação despejada ao usuário, o sistema de busca de informações priorizará pelos serviços mais utilizados ou mais relevantes;
  • Vamos organizar as informações secundárias das categorias num só lugar, aqui é fundamental disponibilizar um ponto de contato rápido com a instituição para situações específicas ou emergenciais;
  • Propor um redesenho de interface.

Na sequência segue uma proposta de navegação no sistema. Cada categoria oferece somente as principais informações. Para acesso a detalhes específicos (navegação mais profunda) disponibilizamos o item "Outros serviços *".

*: Uma pesquisa de campo mostraria quais seriam os itens relacionados destas seções, entretanto, todos estariam disponibilizados conforme indicado.

Projetando experiências e UI

Possibilitando aos usuários uma experiência mobile do site Detran.SP a imagem a seguir mostra as possibilidades de interações na navegação do site e para o sistema de busca.

  1. Menu de navegação: Cada categoria com identificação visual distinta (cor, ícone e texto de apoio). Ao escolher a categoria desejada o menu apresenta os principais serviços. O ambiente cromático da interface sinaliza ao usuário a seção atual de navegação. "Outros serviços" disponibiliza as informaçoes secundárias da seção.
  2. Sistema de busca: Listagem (autocomplete) suscinta, porém relevante ao usuário, mostrando o contexto da aplicação do termo digitado;
  3. Busca por comando de voz: Trata-se de um possibilidade para otimizar a rapidez da busca.

Na versão desktop o menu de navegação proporciona ao usuário encontrar as informações num só lugar. O sistema de busca recebe uma área estratégica na tela e tem mais destaque. Pontos de contato recebem destaques visuais para que o usuário saiba outros meios de encontrar as informações em caso de dúvidas.
